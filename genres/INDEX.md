# Genre library — searchable index

Whole-document structures, keyed by **the reader's goal** (not by topic). To
classify a request, match its goal here, then read the matching
`genres/<name>.md` for the section skeleton. No good fit → pick the closest and
adapt, or define a new genre (see SKILL.md → Governance).

Each entry: `<name> — reader goal · note`.

## Index

- argument — recommend between options or argue for a change, so the reader can
  follow the reasoning, not just the verdict
- explainer — teach how a system or mechanism works; no recommendation
- postmortem — explain what broke and why, after the fact; blameless, timeline-driven
- changelog — list what changed in a release, grouped for fast scanning
